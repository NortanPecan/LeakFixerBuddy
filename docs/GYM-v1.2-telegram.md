# GYM v1.2 — Telegram Integration Tasks

## Цель
Интеграция GYM с Telegram-ботом для:
- Утренних напоминаний с планом
- Вечерних пингов если не отмечено
- Быстрого ввода факта из чата

---

## T1. Утреннее сообщение

**Статус:** 🔲 Pending

**Когда:** Утром (время настраивается) в дни тренировок.

**Текст:**
```
🏋️ GYM сегодня: [Грудь + плечи]

• Жим гантелей 22.5х12х4(25)
• Разводка 15х12х3(17.5)
• Армейский жим 30х10х4(32.5)

[Открыть план] [Перенести] [Отменить]
```

**Кнопки:**
- **Открыть план** → открыть Mini App на экране тренировки
- **Перенести** → диалог переноса
- **Отменить** → диалог пропуска

---

## T2. Вечерний пинг

**Статус:** 🔲 Pending

**Когда:** Вечером (время настраивается) если тренировка не отмечена.

**Текст:**
```
📍 План на сегодня был:

[Грудь + плечи]
• Жим гантелей 22.5х12х4(25)
• ...

Сделал? Отметь или поправь план.
[Да, сделал] [Частично] [Не получилось]
```

**Кнопки:**
- **Да, сделал** → отметить completed, спросить про nextWeight
- **Частично** → открыть Mini App для редактирования
- **Не получилось** → диалог переноса/пропуска

---

## T3. Быстрый ввод факта

**Статус:** 🔲 Pending

**Сценарий:** Пользователь пишет в чат:

```
жим 25х12х4
```

**Бот распознаёт:**
- Упражнение: "Жим" (ищет в шаблонах по названию)
- Вес: 25
- Схема: 12х4

**Ответ:**
```
✅ Жим гантелей: 25кг × 12х4

Следующий раз какой вес?
[Легко +2.5] [Норм] [Тяжело -2.5]
```

---

## T4. Inline-режим

**Статус:** 🔲 Pending (опционально)

**Сценарий:** Пользователь пишет `@bot жим` в любом чате.

**Бот показывает:**
- Последние выполнения упражнения
- Текущий и следующий вес
- Кнопку "Добавить в план"

## Архитектура

### Хранение настроек уведомлений

Добавить в `UserSettings`:
```prisma
model UserSettings {
  // ... существующие поля ...
  gymMorningReminder   Boolean  @default(true)
  gymMorningTime       String   @default("08:00")
  gymEveningReminder   Boolean  @default(true)
  gymEveningTime       String   @default("20:00")
}
```

### Очередь уведомлений

- Таблица `NotificationQueue` или использование существующего механизма
- Cron-задача для проверки и отправки

## Порядок выполнения

1. **T1** — Утреннее сообщение (базовая реализация)
2. **T2** — Вечерний пинг
3. **T3** — Быстрый ввод факта (распознавание текста)
4. **T4** — Inline-режим (опционально)

---

## Зависимости

- T1-T3 требуют работающего Telegram-бота
- T1-T2 требуют API `/api/gym/today`
- T3 требует API для обновления факта и nextWeight

---

## Примечание

Telegram-интеграция — отдельный слой поверх Backend и Frontend.
Реализуется после завершения Backend (B1-B6) и ключевых Frontend-задач (F1-F5).
